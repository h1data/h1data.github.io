ALT-Synthdrum


ダウンロード (maxforlive.com) 最終更新: 2020/5/2, version 1.1.0

シンプルでありがちなモノシンセドラムです。そうです、70年代シンセドラムにインスパイアされています。

使用例

特徴

  • 擬似打撃音を再現しています。ATTACKダイヤルは時間ではなく強さになります。
  • 強力なLFOとSWEEP機構をお楽しみください。
  • RC回路を模した矩形波を使用しています。ノコギリ波もあります。
パラメーター説明
    (以下、オートメーションコントロールセレクタの表示順)
  • MODE: LINEモードとADSRモードを切り替えます。（それぞれ、Maxのline~とadsr~オブジェクトに由来しています）
    LINEモードはノートオンから発音までの遅延がなく、ADSRモードは約1ミリ秒の遅延がありますが再トリガー時のクリックノイズが発生しません。(既定: ADSR)
  • SENSE (Sensitivity): ノートオンヴェロシティに対する音量変化の割合を調整します。
  • B.Tune (Beat Tune): ノートオンヴェロシティに対するピッチ変化量を、オクターブ単位で調整します。
  • VCO (VCO Tuning): ピッチを微調整します。
  • OCTAVE (VCO Octave): ピッチをオクターブ単位で調整します。
  • WAVE (VCO Waveform): VCOの波形を切り替えます。
  • ATTACK: 疑似打撃音の音量を調整します。いわゆるADSRのAttackではないです。
  • SUSTAIN: 音の長さを調整します。
  • LFO (LFO Depth): LFOの深度を調整します。
  • FREQ. (LFO Frequency): LFOの周波数を調整します。
  • WAVE (LFO Waveform): LFOの波形を切り替えます。
  • SWEEP (SWEEP Depth): スィープの深さをオクターブ単位で指定します。
  • DIRECTION (SWEEP Direction): スィープをUP（上方向）、DWN（down、下方向）、およびOFF（なし）のいずれかに切り替えます。
  • OUTPUT (OUTPUT Level): 出力音量を調整します。
既知の問題
  • LINEモードは遅延がありませんが、連続で発音するとクリップノイズが生じることがあります。
  • 常に各波形の発振器 (VCO、LFO)、エンベロープ生成器分の負荷が発生します。

echo on noteoff


Download at ダウンロード (maxforlive.com) 最終更新: 2023/8/27, version 2.0.0

分散和音用の実験的なMIDIエフェクトです。
Arpeggiatorの前段に置くと、アルペジオの音量が減衰していく効果が得られます。

パラメーター説明
    (以下、オートメーションコントロールセレクタの表示順)
  • Delay mode (Sync/Free): ノートオフからエコー発生までの遅延時間をテンポに同期するかミリ秒で指定するかを切り替えます。
  • Delay (free/sync): ノートオフからエコー発生までの遅延時間を調整します。
  • Duration mode (Sync/Free): エコーノートの長さをテンポに同期するかミリ秒で指定するかを切り替えます。
  • Duration(free/sync): エコーノートの長さを調整します。
  • Feedback: エコーのフィードバックの割合を調整します。
  • Velocity: 最初のエコーのヴェロシティを割合で調整します。
  • Reset: 全てのノートを止めます。ボタンを押すたびにLiveの元に戻す履歴が作成されます。
  • Auto Stop: オンの場合、Arpeggiatorに特化した動作モードに切り替わります。
    エコーノートはデバイスがノートオンを受信すると常に停止されるようになり、発音中の実際のノートがある場合はノートオフとエコーノートの発生が保留されるようになります。
既知の問題
  • VSTiやMIDI音源によっては、連続して発音すると直前のMIDIノートのヴェロシティを引き継ぐため、
    エコー効果が得られず同じ音量でノートが継続するだけになったり、新しいMIDIノートのヴェロシティが小さくなる場合があります。
    モノモードやレガートモードでは発音中の別の音階からヴェロシティを引き継いでしまうこともあります。Delayを40msec程に設定すると改善する場合があります。
  • 一度に30音以上同時発音すると、エコーが数ミリ秒ずつ遅れることがあります。

bouncing notes


ダウンロード (maxforlive.com) 最終更新: 2023/8/31, version 1.2.2

*** これは公式のものではありません。もしそちらをお探しの方はhttps://www.ableton.com/packs/へどうぞ。 ***

ノートオンの間、ピンポン玉が跳ねるようにノートが生成されます。視覚的に効果が分かるようアニメーション窓が付いています。
月面の浮遊感から苛烈な60G環境までの跳ね返りをシミュレートできます。
複数ノートの処理や、反転モードに対応しています。

パラメーター説明
  • Gravity: 重力を表し、ボールがどれだけ速く落ちるかを調整します。
  • Velocity: ボールの初速度を調整します。既定値の20は、100*20m/ミリ秒を表し、100の場合は2m/秒になります。
  • Rebound: しばしばeで記される反発係数を表します。係数が大きくなるにつれ、エコーノートのヴェロシティ減少は遅くなります。
  • Vel D/W: ヴェロシティのDry/Wet比を表します。いわゆるヴェロシティセンシティビティです。
  • Duration: ボールがバウンドする間隔に対するノートの長さの比率を調整します。
  • Minimum: ボールがバウンドする間隔の最小時間を調整します。最小時間に達した後の挙動はHOLDボタンのオンオフによって変わります。
  • HOLD: オンの場合、ボールがバウンドする間隔がMinimumで設定された最小時間に達した後も継続します。
    オフの場合、ボールがバウンドする間隔がMinimumで設定された最小時間に達した時点でバウンドが停止します。
  • DISP: アニメーションの表示有無を切り替えます。
  • Inverse: オンの場合、バウンドが反転し、ノートの間隔が徐々に長くなります。
    ヒント: 反転モードでは、Velocityの値を2か3など小さくすることで短い間隔からバウンドが始まって良い感じになります。
既知の問題
  • ノートオンの間、Gravityなどパラメーターを変更しても次のバウンドが始まるまで変更が反映されません。
    この問題に対処するにはパッチの大部分を変更 (Max for Live Essentials のNote Echo の様にCPU時間ごとにイベント管理するなど) する必要がありますが、
    リアルタイムで重力を正確に変えていきたい、といった人は居らっしゃらないと思っています。
  • 一度に大量のノートを演奏するとCPU使用率が上がることがあります。(後段のインストゥルメントによる場合もあります)

PitchW curve


ダウンロード (maxforlive.com) 最終更新: 2025/7/28, version 1.4.0

ノートオンと連動してピッチベンドカーブを出力する簡素なMIDIエフェクトです。
Drum rack内でシンセドラム音色のベンドを下げるのに適しています。
LAEX氏作の"PitchW ADSR"を元に、下記機能を追加しています。

  • ノートオンからピッチベンドカーブ開始までの遅延時間を設定できます。
  • ピッチベンド範囲を14bit精度の値で指定できます。(-8192から8191)
  • ピッチベンド変化は線形だけでなく指数および対数カーブにも対応しています。
  • 左側の窓はピッチベンド値を表示し、上下の枠をドラッグすることでピッチベンド範囲を調整できます。
パラメーター説明
  • From: ノートオン時点の初期ピッチベンド値を調整します。
  • To: ピッチベンドカーブの終着点の値を調整します。
  • Exp. (Exponent): 指数カーブの次数を調整します。
    1.00: 線形、1.00以上: 指数カーブ、1.00以下: 対数カーブになります。
  • Delay mode: ノートオンからカーブ開始までの時間をノート長かミリ秒指定にするかを切り替えます。
  • Delay time: ノートオンからカーブ開始までの時間を調整します。
  • Sustain mode: カーブの持続時間をノート長かミリ秒指定にするかを切り替えます。
  • Sustain time: カーブの持続時間を調整します。
  • Legato: オンにすると、発生中のノートがある間はノートオンを受信してもカーブが始まりません。和音との使用に適しています。（初期値: オフ）
  • ピッチベンド表示部分: ピッチベンドカーブの範囲や現在の値を表示します。上下の枠をドラッグすることで範囲のFrom、Toの値を変更できます。
    Shift を押しながら枠をドラッグすることで微調整ができます。CmdまたはCtrlを押しながらピッチベンド範囲の枠をドラッグすると25段階（上下に12段階と0）で調整できます。
    ShiftとCmdまたはCtrlでは129段階（上下に64段階と0）で調整できます。
使用上の注意と既知の問題
  • ADSRの様に上がってまた戻るカーブには対応していません。オリジナルを試してみてください。
  • 入力されたピッチベンドメッセージはすべて無視されます。
  • ピッチベンドメッセージは1ミリ秒ごとに出力されます。
    ハードウェアのMIDI機器等に対して、もしメッセージの間隔が短すぎる場合は「Interval」でピッチベンドメッセージの送信感覚を調整できます。
  • Shift+ドラッグでピッチベンドカーブの範囲を微調整できますが、マウスポインタの位置はデバイスの表示範囲に留まりません。 Version 1.4.0で改善しました。
  • Max 6では、デバイスをMIDIトラックに新規追加した際に「To」の値が0で表示されますが実際には表示窓のとおり-8192として動作します。

Audio Frame Macro


ダウンロード (maxforlive.com) 最終更新: 2022/10/30, version 1.3.3

PALやNTSCの様なフレームレートで、整数値のパラメーターを粗い解像度で変化させるオーディオエフェクトです。
Trackerやチップチューン用のプラグインに強く影響を受けています。

特徴

  • インストゥルメントデバイスやプラグインのキーシフト、オクターブシフト、波形選択など整数値のパラメーターを調整しやすいよう設計されています。
  • 小数値のパラメーターに対して、スケーリングした値でモジュレーションをかけることも可能です。
  • 4つの独立したモジュレーションに対して別なパラメーターを割り当てることができ、各モジュレーションはデバイスのマスタークロックに同期します。
  • デバイスのMIDI入力にルーティングすることで、MIDIノートを受信することでステップの最初から再開できます。（Live 11以降）
パラメーター説明
  • ClockMode: クロック周期をHzかノート長で指定するかを切り替えます。
  • ClockFrequency: クロック周期をHzで調整します。
    実際のステップ間隔はインターレースフレームを再現するため指定した値の2倍になります。29.97Hz: NTSC規格、25Hz: PAL規格にそれぞれ相当します。
  • CLockRate: クロック周期をノート長で調整します。
  • パラメーターセレクター（Clockの下部）: 4つの独立したパラメーターマクロを切り替えます。
  • Resync: ボタンを押すと、全パラメーターマクロのステップ位置が1番目に戻り同期されます。
    ボタンを押すたびにLiveの元に戻す履歴が作成されます。
    ヒント: Resyncにオートメーションを設定することで、MIDI入力無しでアタック効果を付けることができます。

    • 以下は各パラメーターマクロごとの設定になります。
  • ループセレクター（スライダーの上部）: ループ位置を調整します。
  • スライダー: 各フレームごとの値を調整します。
  • Values（スライダーの下部）: 各フレームごとのモジュレーション値を表示します。ドラッグやキーボード入力で変更でき、オートメーションも設定できます。
    これらの値はInput Rangeの範囲を超えて指定できますが、実際のモジュレーションはInput Rangeに従います。
  • Cycle / Step: スライダーの1ステップに要するクロックのサイクル数を調整します。値が大きくなるにつれステップの変化が遅くなります。
  • Input Range: 各スライダー値の最小値・最大値を調整します。
  • Scale (S): オンの場合、モジュレーション値は対象のパラメーターの範囲にスケーリングされます。
    音量など小数の値を取るパラメーターに適しています。
  • Fit: ボタンを押すと、Input Rangeが対象のパラメーターと同じ範囲に設定されます。
    整数値を取るパラメーターに適しています。モジュレーション対象のパラメーターが設定（Map）されている場合のみ押下できます。
  • 0-7F: ボタンを押すと、Input Rangeが0から127（0x7F）の範囲に設定されます。
    オン・オフや項目値を取るパラメーターが設定（Map）されている場合は押下できません。
  • +/-: ボタンを押すと、Input Rangeが-64から63の範囲に設定されます。パンなど2方向のパラメーターに適しています。
    オン・オフや項目値を取るパラメーターが設定（Map）されている場合は押下できません。
  • Map: ボタンを押すと、マッピングモードに切り替わります。マッピング可能なLive上のパラメーターをクリックしてモジュレーション対象を選択します。
    選択されたパラメーター名はUnmapボタン（x）の右側に表示されます。
  • Unmap (x): ボタンを押すと、マッピングされたパラメーターの選択が解除されます。
既知の問題
  • Clockの周期は、モジュレーションさせるデバイスやLive自体の内部バッファによって、実際の周期とは完全に一致しません。(サンプリング周波数44.1kHzで10～20数サンプル程の誤差)
  • 音量など、値が変化してから数ミリ秒かけて連続的に動作するパラメーターでは、段階的に変化する効果が得られない場合があります。
  • スケーリングしない場合、変化できる最大値は絶対値で128です。十分と考えられますが、これを超える値を持つパラメーターは変化させられません。
    もし128を超える値が持てるデバイス、特にLive標準のデバイスがあればお知らせください。
  • Resyncボタンにもオートメーションが設定可能ですが、副作用としてボタンを押すたびに元に戻す履歴が作成されます。
  • スライダーの下に表示される数値はInput Rangeで指定された範囲を超えることがありますが、モジュレーションに用いられる値はInput Range の範囲に絞られます。

Phototactic XY


ダウンロード (maxforlive.com) 最終更新: 2024/1/28, version 1.0.1

典型的ですが宇宙一不安定なXYパッドです。光に引き寄せられる虫の動きを模しています。

パラメーター
  • Mode: 動き方を切り替えます。
    Step: Intervalの時間ごとに一定距離でランダムな方向に動きます。
    Line: 直線的に動き、Intervalの時間ごとに方向が変わります。
    Curve: ランダムな方向に向かって加速します。このモードでは加速により意図した幅より大きく動くのを抑制するため、Widthの効果は半分になります。
    (各モードでのパラメーター変化のイメージ)
  • Map X / Map Y: クリックしてからパラメーターを選択するとマッピングされます。もう一度押すとマッピングが取り消されます。
  • Fold (▶) / Unfold (▼) : 右側パラメーターの表示・非表示を切り替えます。
  • X-Y Control:
    クリックかドラッグで円マークを動かします。(既定のLiveカラーテーマではオレンジ)
    実際は動いている線 (既定のLiveカラーテーマでは青) の座標によりマッピングされたパラメーターが変化します。
  • X Width: ランダムな動きのX座標幅を指定します。
  • Y Width: ランダムな動きのY座標幅を指定します。
  • Link Width: X座標とY座標の動きの幅を等しくするかどうかを切り替えます。
  • Interval: 動きの間隔を時間で指定します。
    0msにすると毎オーディオサンプルごとに動きますが、多くのデバイスやプラグインでは追随できません。
  • Attract: ランダムな動きがどれだけ円マークに引き寄せられるかの割合を指定します。
  • Position X/Y: 円マークのX座標およびY座標を表します。オートメーションを設定できます。
